This is a port of binarymuse's shouldComponentUpdate demo for React.

Webcomponents are used to implement a React like compenent life cycle, the implementation can be found in the official incremental-dom examples. This allow a real shouldComponentUpdate method that will decide when the component has to patch its own shadow root, istead of an explicit skip of descendants like in the plain primer6 demo.

Unfortunately, outisde of chrome this will require heavy polyfilling (about 100kb) and styling of elements is tricky with webcomponents specification still to come.
